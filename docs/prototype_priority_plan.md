## 2. 1순위: MG-0002 고양이 연금술 공방 MVP

### 2.1 선정 이유

1. **공통 모듈 테스트에 최적**
   - Idle 시스템, Crafting 시스템, 인벤토리 등 핵심 모듈 검증
   - 다른 게임에서 재사용할 수 있는 기반 구축

2. **개발 난이도 낮음**
   - 복잡한 실시간 전투 없음
   - 단순한 터치 인터랙션
   - 서버 의존도 낮음 (오프라인 플레이 가능)

3. **빠른 시장 검증**
   - 2-3개월 내 소프트 론치 가능
   - 힐링/캐주얼 타겟 반응 확인

### 2.2 MVP 범위 정의

#### 포함 기능 (Must Have)

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 재료 수집 | 방치 시간 기반 재료 자동 수집 | P0 |
| 기본 제작 | 레시피 선택 → 제작 → 결과 | P0 |
| 레시피 발견 | 재료 조합으로 새 레시피 해금 | P0 |
| 인벤토리 | 재료/완성품 보관 및 관리 | P0 |
| 판매 시스템 | 완성품 판매 → 골드 획득 | P0 |
| 공방 업그레이드 | 골드로 생산 효율 향상 | P0 |
| 고양이 1마리 | 기본 고양이 캐릭터 (도우미) | P0 |
| 튜토리얼 | 기본 플레이 안내 | P0 |

#### 제외 기능 (Post-MVP)

| 기능 | 이유 | 출시 버전 |
|------|------|-----------|
| 추가 고양이 | 수집 시스템은 MVP 이후 | v1.1 |
| 인테리어 | 꾸미기는 코어 루프 검증 후 | v1.1 |
| 소셜 기능 | 서버 인프라 필요 | v1.2 |
| 가챠 시스템 | 수익화는 리텐션 확인 후 | v1.2 |
| 시즌 이벤트 | 라이브 서비스 단계 | v1.3 |

### 2.3 MVP 기술 스택

```
┌─────────────────────────────────────────────────────────────────┐
│                      기술 스택                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  Frontend                                                       │
│  ├── Flutter 3.x                                               │
│  ├── Flame Engine (게임 렌더링)                                 │
│  ├── Provider/Riverpod (상태 관리)                              │
│  └── mg-common-game (공통 모듈)                                 │
│      ├── IdleSystem                                            │
│      ├── InventorySystem                                       │
│      ├── CraftingSystem (신규 개발)                            │
│      └── UIComponents                                          │
│                                                                 │
│  Backend (MVP는 최소화)                                         │
│  ├── 로컬 저장 (SharedPreferences/Hive)                        │
│  └── 클라우드 백업 (Firebase, 선택적)                           │
│                                                                 │
│  Analytics                                                      │
│  └── mg-common-analytics                                       │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2.4 MVP 개발 일정

| 주차 | 작업 | 산출물 |
|------|------|--------|
| 1-2 | 프로젝트 세팅, 공통 모듈 연동 | 빈 프로젝트 실행 |
| 3-4 | Idle 시스템, 재료 수집 | 방치 재료 수집 동작 |
| 5-6 | Crafting 시스템, 레시피 | 제작 기능 동작 |
| 7-8 | 인벤토리, 판매, 업그레이드 | 코어 루프 완성 |
| 9-10 | UI/UX, 고양이 캐릭터 | 비주얼 완성 |
| 11-12 | 튜토리얼, QA, 버그 수정 | MVP 완성 |

**총 예상 기간**: 12주 (3개월)

### 2.5 MVP 성공 지표

| 지표 | 목표 | 측정 방법 |
|------|------|-----------|
| 기술 검증 | 공통 모듈 3개 이상 검증 | 체크리스트 |
| 코어 루프 | 10분 플레이 가능 | QA 테스트 |
| 리텐션 | D1 40% 이상 | 내부 테스트 |
| 세션 시간 | 평균 5분 이상 | Analytics |

## 3. 2순위: MG-0004 카페 매치 타이쿤 MVP

### 3.1 선정 이유

1. **캐주얼 앵커 역할**
   - 대중적인 Match3 장르로 유저 유입
   - 광범위한 타겟 도달

2. **Match3 엔진 검증**
   - 다른 퍼즐 게임에 재사용 가능
   - 0005, 0009, 0014 등에 활용

3. **검증된 수익 모델**
   - Playrix 모델 참고 가능
   - 광고/IAP 하이브리드

### 3.2 MVP 범위 정의

#### 포함 기능 (Must Have)

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| Match3 코어 | 3개 이상 매칭 → 제거 | P0 |
| 스테이지 20개 | 튜토리얼~초반 난이도 | P0 |
| 목표 시스템 | 스테이지별 클리어 조건 | P0 |
| 하트 시스템 | 플레이 횟수 제한 | P0 |
| 기본 부스터 3종 | 망치, 셔플, +5턴 | P0 |
| 카페 확장 1단계 | 인테리어 3개 해금 | P0 |
| 단골 1명 | 스토리 도입 | P0 |

#### 제외 기능 (Post-MVP)

| 기능 | 이유 | 출시 버전 |
|------|------|-----------|
| 추가 스테이지 | 콘텐츠 확장 | v1.1+ |
| 소셜 기능 | 서버 필요 | v1.2 |
| 시즌 이벤트 | 라이브 서비스 | v1.3 |
| PVP 랭킹 | 서버 필요 | v1.3 |

### 3.3 MVP 개발 일정

| 주차 | 작업 | 산출물 |
|------|------|--------|
| 1-2 | Match3 엔진 개발 | 기본 매칭 동작 |
| 3-4 | 스테이지 시스템, 목표 | 스테이지 플레이 |
| 5-6 | 하트, 부스터, 보상 | 게임 이코노미 |

| 9-10 | 스테이지 20개 제작 | 콘텐츠 완성 |
| 11-12 | 단골 스토리, QA | MVP 완성 |

**총 예상 기간**: 12주 (3개월)
