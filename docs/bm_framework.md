
# Monthly Games BM 설계 프레임워크

> 목적: Monthly Games 24종 라인업에 공통으로 적용할 **BM(수익 모델) 프레임**을 정의하고,  
> 게임별 BM JSON 스키마 설계의 기준 문서로 사용한다.  
> 권장 위치: `mg-meta/docs/business/bm_framework.md`

---

## 0. BM 대전제

- 기본 모델: **F2P + IAP + 리워드 광고 혼합**
- 방향성
  - 과금 = **시간 단축 / 선택지 확장 / 외형 강화**
  - “돈 안 내면 절대 못 깨는” 구간은 만들지 않는다.
  - 월 1게임 구조이므로 **단기 회수용 타이틀 + 장기 라이브용 타이틀**을 나눠 보고,
    - 단기: 패키지/패스/이벤트 중심
    - 장기: 코스튬/시즌 패스/PVP 메타/레이드 중심

---

## 1. 수집형 BM (캐릭터 / 카드 / 용병 등)

### 1.1 적용 대상

- 메인: 0001, 0003, 0006, 0009, 0012, 0013, 0016, 0019, 0020, 0024  
- 서브: 0007, 0011, 0015, 0018 등

### 1.2 무료 구조

- 스토리/튜토리얼/일일 미션에서 기본 캐릭터/카드 지급
- 노멀/레어 급은 무과금 플레이만으로도 수집 가능
- 각 게임마다 실사용 가능한 캐릭터 **최소 10~15개**는 무료 파밍 범위로 둔다.

### 1.3 유료 레버

1. **가챠(뽑기)**  
   - 통화 구조: `소프트 재화 + 프리미엄 재화` 이중 구조  
   - 필수 요소:
     - **천장(보장) 시스템**
     - **픽업 배너** (기간/캐릭터 한정)
   - 밸런스 방침:
     - 수치는 상한선을 두고, 진짜 힘은 **역할/시너지/조합**에서 나오게 설계

2. **확정 선택 패키지**  
   - “N회 뽑기 + 선택권 1장” 형태  
   - 신규/복귀 유저용 진입 패키지로 활용

3. **수집형 스킨/버전**  
   - 동일 캐릭터의 시즌 버전(해변/학원/페스티벌 등)
   - 성능은 동일 또는 소폭 차이, 외형·연출만 큰 차별화

---

## 2. 성장 / 강화 / 전생 BM

### 2.1 적용 대상

- 대부분 게임에 적용, 특히:
  - 0001, 0002, 0003, 0005, 0007~0010, 0013, 0015~0017, 0020, 0023

### 2.2 무료 구조

- 레벨/장비/스킬 업그레이드는 **시간+플레이+방치**로 꾸준히 성장 가능
- 전생/루프 보상은 **플레이 시간/진행도 비례** 지급

### 2.3 유료 레버

1. **성장 재화 패키지**
   - 예: 스타트 대시 패키지, 신규/복귀 부스트 패키지
2. **성장 속도 배수**
   - 월정액/시즌 패스에서 경험치/골드/방치 수익 +X%
3. **전생/리셋 편의**
   - 빌드 초기화 무료 횟수 증가
   - 전생 시 손실 감소, 메타 보상 증가

### 2.4 설계 원칙

- “스펙 그 자체를 돈으로 사는 구조”는 최소화하고,  
  **시간 단축 + 빌드 실험 편의**에 초점을 맞춘다.

---

## 3. 퍼즐 / 스테이지 / 콘텐츠 BM

### 3.1 적용 대상

- 0001, 0002, 0004, 0005, 0009, 0021 등

### 3.2 무료 구조

- 스태미너/에너지 시스템 도입:
  - 캐주얼 유저는 거의 제약을 못 느끼는 수준
  - 광고/IAP로 보충 가능
- 챕터 클리어/별 3개 달성 보상으로 재화 충분 지급

### 3.3 유료 레버

1. **스태미너 패스/부스트**
   - 에너지 최대치, 회복 속도 증가
2. **스테이지 스킵/자동 클리어 티켓**
   - 이미 클리어한 파밍/이벤트 스테이지만 스킵 허용
3. **추가 챌린지 모드**
   - 하드/익스트림 모드 입장 티켓
   - 여기서만 얻는 특별 재화/코스튬 파츠 배치

---

## 4. 방치 수익 / 오프라인 보상 BM

### 4.1 적용 대상

- 0001, 0002, 0003, 0007, 0008, 0010, 0013, 0015, 0017, 0019, 0020, 0022, 0023 등

### 4.2 무료 구조

- 오프라인 수익 상한: 무과금 기준 **8~12시간** 수준
- 접속 안 해도 천천히 성장, 접속하면 훨씬 더 빨리 성장

### 4.3 유료 레버

1. **방치 수익 배수 아이템/패스**
   - 광고 시청으로 2배, 패스로 상시 3배 등
2. **오프라인 상한 확장**
   - 유료 패스로 24시간 이상 누적 가능
3. **리포트 연출 강화(부가 요소)**
   - 전투 리플레이/하이라이트는 유지율 관점에서 고려

---

## 5. PVP / 리그 / 랭킹 / 길드 BM

### 5.1 적용 대상

- 0003, 0006, 0012, 0013, 0016, 0019, 0024 등

### 5.2 무료 구조

- PVP 입장/참여는 무료
- 일일/주간 보상으로 재화/티켓 지급
- 무과금 유저도 상위 20~30%까지 도전 가능하게 밸런싱

### 5.3 유료 레버

1. **시즌 패스 (Battle Pass)**
   - 무료/유료 트랙 분리
   - 유료 트랙 보상: 한정 스킨, 이펙트, 프로필, 이모트 등
2. **리그 전용 코스메틱**
   - 특정 티어 이상 전용 아우라, 칭호, 프레임
3. **길드 스킨/하우징**
   - 길드 깃발, 엠블럼, 길드 하우징 장식

### 5.4 P2W 방지선

- PVP에서 과금을 통해 얻는 이득은  
  **성장 속도/빌드 다양성/외형** 중심으로 제한하고,  
  절대적인 수치 격차는 장기 플레이와 실력 요소에 남겨둔다.

---

## 6. 이벤트 / 레이드 / 시즌 페스티벌 BM

### 6.1 적용 대상

- 0012, 0015, 0019, 0020, 0022, 0024 외 Year1 후반 다수

### 6.2 무료 구조

- 이벤트 스테이지 기본 입장 무료
- 플레이만으로도 이벤트 상점 핵심 보상 70~80% 확보 가능

### 6.3 유료 레버

1. **이벤트 패스**
   - 이벤트 포인트 획득량 증가
   - 한정 스킨/장식/프로필/카드 제공
2. **추가 입장/포인트 부스터**
   - 이벤트 던전 추가 입장권
   - 포인트 2배 부스터
3. **한정 번들**
   - 시즌/테마 패키지(예: 연말 패키지, 여름 패키지 등)

---

## 7. 꾸미기 / 스킨 / 인테리어 BM

### 7.1 적용 대상

- 0002, 0004, 0007, 0010, 0011, 0015, 0017, 0021, 0023 등

### 7.2 무료 구조

- 기본 테마/인테리어/스킨 일부는 스토리/업적/이벤트로 지급
- “개성 표현”은 과금/무과금 모두 접근 가능하게 폭을 넓게 설계

### 7.3 유료 레버

1. **프리미엄 테마 패키지**
   - 카페/숲/왕국 등 장소 단위 테마 세트
   - 세트 효과로 소소한 기능성 버프(1~2% 수준) 검토 가능
2. **한정 스킨/코스튬**
   - 시즌/콜라보 한정
   - 성능 동일 or 거의 동일, 외형/연출 차별화
3. **UGC 확장권**
   - 슬롯 확장, 프리셋 저장 수 증가, 색상 팔레트 추가 등

## 8. 광고 BM (Reward / Interstitial / Offerwall)

### 8.1 기본 방침

- 광고는 “조금 더 얻고 싶을 때 선택하는 옵션”으로 설계
- 강제 전면광고는 최소화, 필요 시 “광고 제거 패스” 제공

### 8.2 형태별 활용

1. **리워드 광고**
   - 방치 수익 2배
   - 실패 후 추가 기회
   - 무료 추가 뽑기/스태미너/이벤트 포인트
2. **인터스티셜(중간 전면)**
   - 매우 짧은 세션 중심 게임(미니게임/라이트 퍼즐)에서만 제한적으로 사용
3. **오퍼월**
   - 특정 지역/플랫폼에서만 적용
   - 고가 재화를 미션형 광고 보상으로 지급

## 9. 미니게임 / 라이브이벤트 BM

### 9.1 적용 대상

- 메인: 0022  
- 서브: 0004, 0007, 0011, 0024 등

### 9.2 무료 구조

- 미니게임 대부분은 무료 입장 + 일일 보상 상한
- 기본 참여만으로도 소소한 보상 확보 가능

### 9.3 유료 레버

1. **추가 시도권/도전권**
   - 랭킹/하이스코어용 추가 플레이
2. **이벤트 아바타/프레임/이모트**
   - 미니게임 전용 시즌 보상
3. **미니게임 공용 부스트**
   - 여러 미니게임에서 공통으로 적용되는 포인트/보상 부스터

## 10. BM JSON 스키마 예시

각 게임별 BM 설계는 다음과 같은 JSON 구조를 기준으로 정의한다.  
권장 파일명: `mg-game-000X/docs/bm_game_000X.json`

```json
{
  "game_id": "game_0001",
  "bm_model": {
    "pricing_tier": "f2p_iap_ads",
    "main_revenue_sources": [
      "character_gacha",
      "growth_packages",
      "event_pass"
    ],
    "elements": {
      "character_collection": {
        "type": "gacha+select_pack",
        "has_pity": true,
        "notes": "서브컬처 라인 공통 캐릭터 풀 사용"
      },
      "growth": {
        "type": "time_skip+boosters",
        "subscription": "monthly_boost_pass"
      },
      "puzzle_stage": {
        "stamina": {
          "base_cap": 60,
          "refill_minutes": 5,
          "monetization": ["iap_refill", "ad_refill"]
        },
        "skip_ticket": {
          "available_from": "chapter_3",
          "monetization": ["drops", "events", "iap_bundle"]
        }
      },
      "idle_income": {
        "offline_cap_hours_free": 12,
        "offline_cap_hours_paid": 24,
        "boost": ["ad_x2", "pass_x3"]
      },
      "ads": {
        "rewarded": true,
        "interstitial": false
      }
    }
  }
}
```

이 스키마를 공통 규격으로 삼고,  
각 게임의 BM 차별점은 `main_revenue_sources`와 `elements` 안의 상세 설정으로 표현한다.
