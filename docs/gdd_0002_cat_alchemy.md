## 1. 게임 개요

### 1.1 기본 정보

| 항목 | 내용 |
|------|------|
| 타이틀 | 고양이 연금술 공방 (Cat Alchemy Workshop) |
| 장르 | Crafting + Idle + 힐링 시뮬레이션 |
| 플랫폼 | iOS, Android |
| 타겟 유저 | 20-40대 여성, 힐링/캐주얼 게임 선호자 |
| 세션 길이 | 5-15분 |
| 수익 모델 | IAP (고양이 가챠, 인테리어) + 광고 (선택적) |

### 1.2 한 줄 설명

> "귀여운 고양이와 함께 신비로운 연금술 아이템을 만들어 판매하는 힐링 공방 시뮬레이션"

### 1.3 핵심 경험 (Core Fantasy)

- 나만의 아늑한 공방에서 고양이와 함께하는 힐링
- 새로운 레시피를 발견하는 실험의 즐거움
- 점점 성장하는 공방을 보는 성취감

## 2. 게임 루프

### 2.1 코어 루프

```
┌─────────────────────────────────────────────────────────────────────┐
│                         코어 게임 루프                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│    ┌──────────┐     ┌──────────┐     ┌──────────┐     ┌──────────┐ │
│    │ 재료 수집 │ ──→ │  제작    │ ──→ │  판매    │ ──→ │ 업그레이드│ │
│    └──────────┘     └──────────┘     └──────────┘     └──────────┘ │
│         ↑                                                    │     │
│         └────────────────────────────────────────────────────┘     │
│                                                                     │
│    세부 흐름:                                                       │
│    1. 재료 수집: 방치 시간 기반 자동 생성 + 터치 수집               │
│    2. 제작: 레시피 선택 → 재료 투입 → 결과물 생성                   │
│    3. 판매: 완성품을 상점에 진열 → NPC 구매 → 골드 획득            │
│    4. 업그레이드: 골드로 공방/장비/고양이 강화                      │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 2.2 세션 설계

| 세션 유형 | 시간 | 주요 활동 |
|-----------|------|-----------|
| 짧은 세션 | 1-3분 | 재료 수령, 제작 시작, 완성품 판매 |
| 일반 세션 | 5-10분 | 레시피 실험, 공방 관리, 고양이 상호작용 |
| 긴 세션 | 15-30분 | 대량 제작, 인테리어 변경, 도감 확인 |

### 2.3 방치 보상 설계

```
오프라인 시간에 따른 보상:

0-1시간: 기본 재료 × 시간(분)
1-4시간: 기본 재료 × 60 + 고급 재료 × (시간-1) × 10
4-8시간: 위 + 희귀 재료 × (시간-4) × 2
8시간+: 8시간 상한 (무한 축적 방지)

※ 고양이 레벨에 따라 배율 증가
※ 인테리어 보너스 적용
```

## 3. 핵심 시스템

### 3.1 재료 시스템

#### 3.1.1 재료 등급

| 등급 | 색상 | 획득 방법 | 예시 |
|------|------|-----------|------|
| 일반 (Common) | 회색 | 자동 생성 | 물, 풀, 흙 |
| 고급 (Uncommon) | 초록 | 느린 자동 생성, 퀘스트 | 은방울꽃, 달빛이슬 |
| 희귀 (Rare) | 파랑 | 매우 느린 생성, 이벤트 | 유니콘 눈물, 용의 비늘 |
| 전설 (Legendary) | 보라 | 특별 이벤트, 업적 | 현자의 돌 조각, 시간의 모래 |

#### 3.1.2 재료 저장소

```
┌─────────────────────────────────────┐
│           재료 저장소               │
├─────────────────────────────────────┤
│  기본 용량: 100 슬롯               │
│  확장: 골드/젬으로 +20씩 확장      │
│  최대: 500 슬롯                    │
│                                    │
│  자동 정리: 등급별/종류별 정렬     │
│  빠른 판매: 일반 재료 일괄 판매    │
└─────────────────────────────────────┘
```

### 3.2 제작 시스템

#### 3.2.1 제작 프로세스

```
1. 레시피 선택
   ├── 해금된 레시피 목록에서 선택
   └── 또는 "실험 모드"로 새 조합 시도

2. 재료 투입
   ├── 필요 재료 자동 표시
   ├── 보유량 확인
   └── "제작" 버튼 활성화

3. 제작 진행
   ├── 제작 시간 (30초 ~ 5분)
   ├── 고양이 도움 시 시간 단축
   └── 즉시 완료 (광고/젬)

4. 결과 확인
   ├── 일반 성공 (100%)
   ├── 대성공 (품질 150%, 확률 10-20%)
   └── 실패 (재료 50% 반환, 확률 5-10%)
```

#### 3.2.2 레시피 구조

```dart
class Recipe {
  String id;
  String name;
  String description;
  RecipeCategory category;

  List<MaterialRequirement> materials;  // 필요 재료
  int craftingTime;                     // 제작 시간 (초)
  int basePrice;                        // 기본 판매가

  int requiredLevel;                    // 해금 레벨
  bool isDiscovered;                    // 발견 여부
  int craftCount;                       // 제작 횟수 (숙련도)
}

class MaterialRequirement {
  String materialId;
  int quantity;
}
```

#### 3.2.3 레시피 카테고리

| 카테고리 | 설명 | 예시 아이템 |
|----------|------|-------------|
| 물약 (Potion) | 효과가 있는 액체류 | 체력 물약, 마나 물약, 사랑의 묘약 |
| 음식 (Food) | 먹을 수 있는 것 | 마법 쿠키, 요정 케이크 |
| 장신구 (Accessory) | 착용 아이템 | 행운의 팔찌, 보호 목걸이 |
| 도구 (Tool) | 사용 아이템 | 마법 빗자루, 투명 망토 |
| 재료 (Material) | 고급 재료 제작 | 정제된 은, 마력 결정 |

#### 3.2.4 레시피 발견 시스템

```
발견 모드:
┌─────────────────────────────────────────────────────────────────┐
│  "어떤 조합이 될까요?"                                          │
│                                                                 │
│  ┌─────┐  ┌─────┐  ┌─────┐                                     │
│  │ ??? │  │ ??? │  │ ??? │   ← 재료 슬롯 (1-3개)               │
│  └─────┘  └─────┘  └─────┘                                     │
│                                                                 │
│  힌트: "붉은 재료와 푸른 재료를 섞으면..."                      │
│                                                                 │
│  [실험하기] 버튼                                                │
│                                                                 │
│  결과:                                                          │
│  - 성공: 새 레시피 해금! + 첫 제작                             │
│  - 실패: "이 조합은 맞지 않는 것 같아요" + 재료 50% 반환       │
│  - 근접: "뭔가 있는 것 같아요! 조금만 바꿔보세요"              │
└─────────────────────────────────────────────────────────────────┘
```

### 3.3 고양이 시스템

#### 3.3.1 고양이 역할

| 역할 | 효과 | 발동 조건 |
|------|------|-----------|
| 재료 수집 도우미 | 자동 수집량 +10~50% | 상시 |
| 제작 보조 | 제작 시간 -10~30% | 제작 시 |
| 대성공 촉진 | 대성공 확률 +5~15% | 제작 시 |
| 손님 유치 | 판매 수익 +5~20% | 판매 시 |
| 힐링 파트너 | 터치 시 귀여운 반응 | 상호작용 시 |

#### 3.3.2 고양이 속성

```dart
class Cat {
  String id;
  String name;
  CatRarity rarity;          // Common, Rare, Epic, Legendary
  CatPersonality personality; // 성격: 활발, 느긋, 호기심 등
  CatSpecialty specialty;     // 전문: 물약, 음식, 장신구 등

  int level;                 // 1-100
  int experience;
  int affection;             // 호감도 0-100

  Map<String, double> bonuses; // 보너스 효과
}

enum CatRarity {
  common,    // 일반 (50%)
  rare,      // 희귀 (30%)
  epic,      // 에픽 (15%)
  legendary  // 전설 (5%)
}
```

#### 3.3.3 기본 고양이 (MVP)

| 이름 | 희귀도 | 전문 분야 | 특수 능력 |
|------|--------|-----------|-----------|
| 먼지 (Dusty) | Common | 균형형 | 모든 보너스 +5% |

*추가 고양이는 v1.1 이후 가챠로 획득*

│                        공방 상점                                │
├─────────────────────────────────────────────────────────────────┤

│  │물약A│ │물약B│ │쿠키 │ │  -  │ │  -  │ │  -  │              │

│  └─────┘ └─────┘ └─────┘ └─────┘ └─────┘ └─────┘              │

│  - 특정 손님은 특정 아이템 선호                                │

#### 3.5.2 가격 책정

```
기본 가격 = 재료 비용 합계 × 1.5 + 제작 난이도 보정

품질 보정:
- 일반: ×1.0
- 대성공: ×1.5

숙련도 보너스:
- 10회 제작: +5%
- 50회 제작: +10%
- 100회 제작: +15%
```

│  │         ↓               ↓               ↓              │   │
│  │      터치→수집      터치→제작      터치→판매           │   │
│  │                                                         │   │
│  │                    🐱 고양이                            │   │

```

### 5.2 주요 화면 플로우

```
메인 화면 (공방)
├── 재료함 터치 → 재료 목록 팝업
│   └── 재료 상세 → 사용처 레시피 보기
├── 가마솥 터치 → 제작 화면
│   ├── 레시피 선택
│   ├── 재료 확인
│   └── 제작 시작/완료
├── 진열대 터치 → 상점 관리
│   ├── 아이템 진열
│   └── 판매 현황
├── 고양이 터치 → 상호작용
│   ├── 쓰다듬기
│   ├── 간식 주기
│   └── 상태 확인
└── 하단 메뉴
    ├── 레시피북 → 레시피 목록/발견
    ├── 상점 → IAP/광고 보상
    ├── 도감 → 수집 현황
    └── 설정 → 옵션/계정
```

### 5.3 터치 피드백

| 액션 | 피드백 |
|------|--------|
| 재료 수집 | 파티클 이펙트 + 팝업 숫자 (+3) |
| 제작 시작 | 가마솥 연기 애니메이션 |
| 제작 완료 | 빛나는 이펙트 + 팡파레 사운드 |
| 대성공 | 황금빛 이펙트 + 축하 메시지 |
| 판매 | 동전 이펙트 + 캐싱 사운드 |

class IdleSystem {
  // 오프라인 보상 계산
  IdleReward calculateOfflineReward(Duration offlineTime, PlayerData player) {
    final cappedTime = min(offlineTime, Duration(hours: 8));
    final minutes = cappedTime.inMinutes;

    final baseRate = getBaseRate(player.workshopLevel);
    final catBonus = getCatBonus(player.activeCat);
    final decorBonus = getDecorationBonus(player.decorations);

    final materials = generateMaterials(
      minutes: minutes,
      baseRate: baseRate,
      bonusRate: catBonus + decorBonus,
    );

    return IdleReward(materials: materials, duration: cappedTime);
  }

  // 실시간 자동 수집 (접속 중)
  Stream<Material> startAutoCollection(PlayerData player) {
    return Stream.periodic(Duration(seconds: 10), (_) {
      return generateRandomMaterial(player);
    });
  }
}
```

### 8.3 제작 시스템 구현

```dart
class CraftingSystem {
  CraftingResult craft(Recipe recipe, PlayerData player) {
    // 1. 재료 확인
    if (!hasEnoughMaterials(recipe, player)) {
      return CraftingResult.insufficientMaterials();
    }

    // 2. 재료 소모
    consumeMaterials(recipe, player);

    // 3. 결과 결정
    final random = Random();
    final successRate = getSuccessRate(recipe, player);
    final criticalRate = getCriticalRate(recipe, player);

    if (random.nextDouble() > successRate) {
      // 실패: 재료 50% 반환
      returnMaterials(recipe, player, 0.5);
      return CraftingResult.failure();
    }

    if (random.nextDouble() < criticalRate) {
      // 대성공
      return CraftingResult.critical(recipe.product, quality: 1.5);
    }

    // 일반 성공
    return CraftingResult.success(recipe.product);
  }
}
```

### 8.4 공통 모듈 연동

```dart
// mg-common-game 연동
import 'package:mg_common_game/core/engine/game_manager.dart';
import 'package:mg_common_game/core/engine/event_bus.dart';
import 'package:mg_common_game/core/engine/scene_manager.dart';

class CatAlchemyGame extends FlameGame {
  late final GameManager _gameManager;
  late final EventBus _eventBus;
  late final IdleSystem _idleSystem;      // 신규 개발
  late final CraftingSystem _craftingSystem; // 신규 개발
  late final InventorySystem _inventorySystem; // 신규 개발

  @override
  Future<void> onLoad() async {
    _eventBus = GetIt.I<EventBus>();
    _gameManager = GameManager(_eventBus);

    _idleSystem = IdleSystem();
    _craftingSystem = CraftingSystem();
    _inventorySystem = InventorySystem();

    // 오프라인 보상 처리
    final reward = _idleSystem.calculateOfflineReward(
      getOfflineDuration(),
      playerData,
    );
    if (reward.hasMaterials) {
      showOfflineRewardPopup(reward);
    }

    // 자동 수집 시작
    _idleSystem.startAutoCollection(playerData).listen((material) {
      _inventorySystem.addMaterial(material);
      _eventBus.emit(MaterialCollectedEvent(material));
    });
  }
}
```
